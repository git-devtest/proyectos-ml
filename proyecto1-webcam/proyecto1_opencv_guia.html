¿Qué aprenderás en este proyecto?

🖼️ Imagen = Matriz numérica

Una imagen es un arreglo 3D de números: alto × ancho × 3 canales. Cada píxel tiene 3 valores (0–255) para Blue, Green y Red.

🔵 BGR, no RGB

OpenCV usa el orden B-G-R por razones históricas. Si ves colores raros, asumiste RGB. Usa cv2.cvtColor() para convertir.

🎞️ Frame = foto instantánea

Un video es una secuencia de frames. Con cap.read() capturas uno por ciclo del loop infinito while True.

🔢 Filtros = operaciones matemáticas

Cada filtro transforma la matriz de píxeles: suaviza promediando vecinos, detecta bordes calculando gradientes, etc.

Nota clave: En numpy, las imágenes se indexan como img[y, x] — primero fila (eje Y), luego columna (eje X). Al principio confunde porque en matemáticas se escribe (x, y). Recuerda: primero filas, luego columnas.

🔢 Teclado numérico (numpad) en Linux + OpenCV

Problema: las teclas del numpad no cambiaban el filtro aunque las teclas superiores sí funcionaban.

Causa: con backend GTK en Linux, el numpad envía códigos distintos. El offset es 176 + dígito (no el código ASCII estándar).

Solución aplicada: usar in (ord('N'), 176+N) para aceptar ambos teclados simultáneamente.
Tecla superior Código Numpad Código Filtro

250Numpad 2178Grises
351Numpad 3179Blur
452Numpad 4180Bordes

Archivo principal

Estructura: Todo el proyecto vive en un solo archivo. Lee el código de arriba hacia abajo — primero la cámara, luego el loop principal, luego los filtros. Cada sección está numerada con comentarios.
proyecto1_webcam.py python 
import cv2
import numpy as np

# ── 1. Abrir cámara (índice 0 = primera cámara USB) ──────────────
cap = cv2.VideoCapture(0)

if not cap.isOpened():
    print("Error: no se pudo abrir la cámara")
    exit()

# ── 2. Selector de filtro activo ──────────────────────────────────
filtro_activo = "original"

# ── 3. Loop principal ─────────────────────────────────────────────
while True:
    # Leer un frame de la cámara
    ret, frame = cap.read()
    if not ret:
        print("Error al leer frame")
        break

    # ── 4. Aplicar filtro según selección ─────────────────────────
    if filtro_activo == "gris":
        resultado = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

    elif filtro_activo == "blur":
        resultado = cv2.GaussianBlur(frame, (15, 15), 0)

    elif filtro_activo == "bordes":
        gris = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)
        resultado = cv2.Canny(gris, 100, 200)

    elif filtro_activo == "sepia":
        kernel = np.array([[0.272, 0.534, 0.131],
                             [0.349, 0.686, 0.168],
                             [0.393, 0.769, 0.189]])
        resultado = cv2.transform(frame, kernel)
        resultado = np.clip(resultado, 0, 255).astype(np.uint8)

    elif filtro_activo == "invertido":
        resultado = cv2.bitwise_not(frame)

    else:  # "original"
        resultado = frame

    # ── 5. Mostrar información en pantalla ────────────────────────
    cv2.putText(
        resultado,
        f"Filtro: {filtro_activo}  |  1-6 cambiar  |  Q salir",
        (10, 30),
        cv2.FONT_HERSHEY_SIMPLEX,
        0.6,
        (255, 255, 0),
        2
    )

    # ── 6. Mostrar ventana ────────────────────────────────────────
    cv2.imshow("Proyecto 1 - Webcam", resultado)

    # ── 7. Capturar teclado ───────────────────────────────────────
    tecla = cv2.waitKey(1) & 0xFF

    if   tecla == ord('q'): break
    elif tecla == ord('1'): filtro_activo = "original"
    elif tecla == ord('2'): filtro_activo = "gris"
    elif tecla == ord('3'): filtro_activo = "blur"
    elif tecla == ord('4'): filtro_activo = "bordes"
    elif tecla == ord('5'): filtro_activo = "sepia"
    elif tecla == ord('6'): filtro_activo = "invertido"

# ── 8. Liberar recursos ───────────────────────────────────────────
cap.release()
cv2.destroyAllWindows()
▶ Ejecutar: python3 proyecto1_webcam.py ⏹ Salir: tecla Q 🎨 Cambiar filtro: teclas 1–6

¿Qué hace cada sección clave?

cap.read() → (ret, frame)

Retorna dos valores: ret es True si leyó bien, frame es el numpy array de la imagen. Si ret es False, la cámara se desconectó.

cv2.waitKey(1) & 0xFF

Espera 1ms por una tecla. El & 0xFF garantiza compatibilidad en sistemas de 64 bits aislando el byte bajo del código de tecla.

cap.release()

Libera el hardware de la cámara. Sin esto, el dispositivo queda bloqueado y no podrás usarlo en otra aplicación hasta reiniciar.

cv2.destroyAllWindows()

Cierra todas las ventanas de OpenCV. Importante llamarlo al final o las ventanas pueden quedar "fantasma" en tu escritorio.

Selecciona un filtro para ver cómo funciona internamente

1 — Original
frame (sin modificar)
Frames crudos tal como vienen de la cámara. Referencia base.
2 — Escala de grises
cv2.cvtColor(frame, COLOR_BGR2GRAY)
Combina 3 canales en 1 usando pesos perceptuales del ojo humano.
3 — Desenfoque Gaussiano
cv2.GaussianBlur(frame, (15,15), 0)
Promedia cada píxel con sus vecinos usando distribución gaussiana.
4 — Detección de bordes (Canny)
cv2.Canny(gris, 100, 200)
Detecta cambios bruscos de intensidad. Algoritmo de 4 pasos.
5 — Filtro Sepia
cv2.transform(frame, kernel_3x3)
Multiplicación matricial que mezcla canales para tono vintage.
6 — Invertido (negativo)
cv2.bitwise_not(frame)
Resta cada valor de 255: oscuro↔claro, colores complementarios.

✅ Retos A, B, C, D y E completados. Cada reto fue commiteado de forma atómica en el repositorio. Ver historial con git log --oneline.
A

Guardar captura con tecla "S"

Agrega un elif tecla == ord('s') que llame a cv2.imwrite('captura.png', resultado). Guarda el frame actual como imagen PNG en el directorio actual.

B

Mostrar dimensiones del frame en pantalla

Usa frame.shape para obtener (alto, ancho, canales) y muéstralo con un segundo cv2.putText() en la esquina inferior izquierda.

C

Mostrar la imagen en espejo

Agrega frame = cv2.flip(frame, 1) justo después de cap.read(). El parámetro 1 = volteo horizontal. Cambia a 0 para volteo vertical.

D

Dibujar un rectángulo centrado

Usa cv2.rectangle(resultado, pt1, pt2, (0,255,0), 2). Calcula el centro con frame.shape[1]//2 (ancho) y frame.shape[0]//2 (alto). El rectángulo debería moverse si el frame cambia de resolución.

E

Filtro "pixelado" (más difícil)

Reduce la imagen con cv2.resize(frame, (64, 48)) y luego vuelve al tamaño original con cv2.resize(..., (ancho, alto), interpolation=cv2.INTER_NEAREST). Resultado: efecto 8-bit retro.
